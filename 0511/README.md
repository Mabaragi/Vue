### Server & Client

Server
- 클라이언트에게 정보와 서비스를 제공하는 컴퓨터 시스템
- 서비스 전체를 제공하는경우 : 장고 웹 서비스
  - 장고를 통해 전달된 HTML 에는 웹 페이지를 구성 할 수 있는 모든 데이터가 포함된
- 정보를 제공하는 경우: DRF API Service
  - 장고로 관리하는 정보만을 JSON 형태로 전달

Client
- 서버가 제공하는 서비스에 적절한 요청을해 응답을 사용자에게 표현하는 기능을 가진 프로그램(웹브라우저 등)

### Cross-Origin ResourCe Sharing
- 브라우저가 요청을 보내고 서버의 응답이 브러우저에 도착
- server는 정상적으로 응답했지만 브라우저가 막은것
- 보안성의 이유로 브라우저는 동일출처정책에 의해 다른 출처의 리소스와 상호작용 하는것을 제한함

SOP
- URL의 Protocol, Host, Port를 모두 포함하여 출처라고 부름
- Protocol://Host:Port/ 가 모두 같아야됨

CORS 교차출처 리소스공유
- 추가 HTTP Header를 사용하여 다른 출처의 자원에 접근 할 수 있는 권한을 부여하도록 할 수 있음
- 다른 출처의 리소스를 가져오기 위해서 서버가 브라우저에게 다른 출처지만 접근해도 된다는 사실을 알리는것

django-cors-headers
