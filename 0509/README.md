### INTRO
우리는 비슷한 것끼리 묶거나(categorizing) 내용을 구성해서 인지하는 것이 편하다는 것을 알고있다. 이러한 요소들은 유저와 밀접한 부분이기에 모든 서비스에서 반드시 고려 되어야한다. 단순한 느낌이나 심미적인 부분만 고려한느것이 아닌 **체계적인 설계**를 통해 기획해야한다

### UX
User Experience
- 유저와 가장 가까이에 있는 분야
- 데이터를 기반으로 유저를 조사하고 분석해서 개발자, 디자이너가 이해 할수 있게 소통
- 유저가 느끼는 느낌, 태도, 행동을 디자인
- 사람들의 마음과 생각을 이해하고 우리 제품에 녹여내는 과정이필요
- 유저 리서치, 데이터 설계 및 정제, 유저 시나리오, 프로토타입 설계 등 필요

좋은 UI 설계

디자이너, 기획자, 개발자
- 개발자는 기능개발만 하는 사람이 아니며 제품에 대해 고민하고 소통하는 능력이 반드시 필요
- 다양한 분야와 협업이 필수이기에 개발자에게 UI, UX 공부는 필수적임

### Routing
- 네트워크에서 경로를 선택하는 프로세스
- 웹에서는 유저가 방문한 URL에 대해 적절한 결과를 응답하는것

SSR
- Server Side Rendering
- URL로 요청이 들어오면 완성된 HTML을 제공하는 방식
- Routing 권한을 서버가 가짐

Routing in SPA/ CSR
- 서버는 하나의 HTML만을 제공함
- JavaScript, axios, ajax 등을 활용

routing이 필요한 이유
- 유저의 사용성 관점에서는 필요함
- 유저가 url을 통해 페이지의 변화 감직 가능
- 새로고침시 처음 페이지로 돌아가는것, 링크를 공유할시 처음페이지만 공유 가능한것, 뒤로 가기를 사용하지 못하는 등등 불편한일발생

Vue Router
- vue의 공식 Router
- SPA 상에서 라우팅을 쉽게 개발 할 수 있는 기능을 제공(일종의 라이브러리)
- 라우트에 컴포넌트를 매핑한 후 어떤 URL에서 렌더링 할지 알려줌
- SPA의 단점 중 하나인 "URL"이 변경되지 않는것을 해결

history mode
- 브라우저의 history API를 활용한 방식
- 새로고침없이 URL 이동기록을 남길 수 있음
- 우리에게 익숙한 URL 구조로 사용 가능
- 사용하지 않으면 뒤로가기 등이 불가능?

router-link
- a 태그와 비슷한 기능.(URL이동시킴)
- routes에 등록된 컴포넌트와 매핑됨
- 히스토리 모드에서는 클릭 이벤트를 차단하여 페이지를 다시 로드하지 않도록함
- 목표 경로는 `to` 속성으로 지정됨.
- a 태그로 렌더링 되지만 필요에따라 다른 태그로 사용 가능

router-view
- 주어진 URL에 대해 일치하는 컴포넌트를 렌더링하는 컴포넌트
- 실제 컴포넌트가 DOM에 부착되어 보이는 자리를 의미
- router-link를 클릭하면 routes에 매핑된 컴포넌트를 렌더링
- Django 에서 block 태그와 비슷함
- app.vue는 일종의 base.html 역할
  
router/index.js
- 라우터에 관련된 정보 설정이 작성되는곳.
- django의 urls.py와 비슷

Views(폴더별 컴포넌트 배치)
- views/
  - routes에 매핑되는 컴포넌트
  - 다른 컴포넌트와 구분하기위해 View로 끝나도록 만드는것을 권ㄴ장
- components/
  - routes에 매핑된 컴포넌트의 하위 컴포넌트를 모아두는 폴도
  - ex) HomeView 컴포넌트 내부의 HelloWorld 컴포넌트

### 선언적 네비게이션과 프로그래밍 네비게이션

선언적 방식 네비게이션
- router-links의 `to` 속성으로 전달하는 방식

Named Routes
- 이름을 가지는 routes

프로그래밍 방식 네비게이션
- vue 인스턴스 내부에서 라우터 인스턴스에 $router로 접근 할 수 있음
- 다른URL로 이동하려면 this.$router.push를 사용
- 어떤 행동을 했을때 이동시키는 방식(redirect 등)
  - history stack에 이동할 url을 넣는 방식
  - history stack에 기록이 남기 때문에 사용자가 브라우저의 뒤로가기 버튼을 클릭하면 이전 url로 이동 할 수 있음
- router-link :to="..."를 클릭하는것과 push(...)는 동일한 동작
