### 프론트엔드 개발이란?
개요
- 프론드 엔드 개발은 웹앱 또는 웹 앱사이트의 UI/UX를 제작하고 관리하는 과정
- 프론트 엔드 프레임워크와 라이브러리를 사용하여 개발 효율성을 높이고 웹앱 복잡성을 관리
- 프론트 개발에서 주요 기술은 HTML, CSS, JSS
  
웹앱
- 웹브라우저에서 실행되는 어플리케이션 소프트웨어
- 웹페이지가 디바이스에 따라서 설치된 App 처럼 보임

SPA (Single Page Application)
- 웹엡에서 자주 등장하는용여.
- 사용자의 요청에따라 적절한 페이지 template을 반환 하는것이 아닌 최초 1장의 HTML만 전달받아 모든 요청에 대응하는 방식
- CSR(Client Side Rendering) 방식으로 모든 요청에 대비 할 수 있음.
- 네이버는 MPA, 네이버 지도는 SPA

SSR(Sever Side Rendering) 이란?
- 서버가 사용자의 요청에 적합한 HTML을 렌더링하여 제공하는 방식. 새로고침수반

CSR
- 최초 한장의 HTML을 받아옴. 서버로 오는것은 거의 비어있는 HTML
- 각 요청에 대한 대응을 JavaScript를 사용하여 필요한 부분만 다시 렌더링
- 필요한 페이지를 서버에 AJAX로 요청
- 서버는 화면을 그리기 위해 필요한 데이터를 JSON 방식으로 전달, JSON 데이터를 JavaScript로 처리, DOM트리에 반영

CSR의 장점
- 모든 HTML 을 받아서 표시하지 않아도 되므로 트래픽감소 > 응답속도 향상
- 매번 새로고침을 하는것이 아니므로 요청이 끊김없이 진행 > UX 향상
- BE와 FE의 작업 영역을 명확히 분리 할 수 있음 > 협업 용이

CSR의 단점
- 첫 구동시 필요한 데이터가 많으면 많을수록 최초 작동 시작까지 오랜시간이 소요
-  모바일에 설치된 Web-App을 실행하게되면 로딩시간이 필요
